# 게임 인터페이스 개선 가이드

## 현재 상태

Unity 게임이 없어도 게임처럼 보이도록 2D 캔버스 기반 게임 뷰를 구현했습니다.

## 구현된 기능

### 1. 2D 게임 캔버스 (`components/game/character-game-canvas.tsx`)
- ✅ 게임 월드 (집, 마당, 나무, 꽃)
- ✅ 캐릭터 표시 및 이동
- ✅ 이벤트 발생 시 캐릭터가 플레이어에게 이동
- ✅ 대화 시스템 시각화
- ✅ 게임 이벤트 애니메이션

### 2. 게임 플레이스홀더 개선
- ✅ characterData가 있으면 2D 게임 캔버스 표시
- ✅ characterData가 없으면 기본 플레이스홀더 표시

## 게임처럼 보이게 하기 위한 추가 작업

### 1. 캐릭터 애니메이션 강화
- [ ] 걷기 애니메이션 (이동 중)
- [ ] 대기 애니메이션 (idle)
- [ ] 대화 애니메이션 (말하기)
- [ ] 감정 표현 애니메이션

### 2. 게임 월드 확장
- [ ] 더 많은 장식 요소 (돌, 풀, 꽃 등)
- [ ] 계절별 배경 변경
- [ ] 시간대별 배경 변경 (낮/밤)
- [ ] 날씨 효과 (비, 눈 등)

### 3. 게임 UI 개선
- [ ] 미니맵 추가
- [ ] 퀘스트 목록 오버레이
- [ ] 인벤토리 UI
- [ ] 설정 메뉴

### 4. 게임 이벤트 시각화
- [ ] 이벤트 발생 시 파티클 효과
- [ ] 포인트 획득 시 숫자 표시
- [ ] 레벨업 시 특수 효과
- [ ] 배지 획득 시 애니메이션

### 5. 사운드 효과
- [ ] 배경 음악
- [ ] 효과음 (이동, 대화, 이벤트 등)
- [ ] 사운드 설정 UI

### 6. 캐릭터 상호작용
- [ ] 캐릭터 클릭 시 상세 정보 표시
- [ ] 캐릭터와 대화하기
- [ ] 캐릭터에게 아이템 주기
- [ ] 캐릭터 상태 확인

## 즉시 적용 가능한 개선 사항

### 1. 캐릭터 스프라이트 추가
현재는 CharacterAvatar 컴포넌트를 사용하고 있지만, 더 게임처럼 보이려면:
- 캐릭터 스프라이트 시트 준비
- 걷기, 대기, 대화 애니메이션 프레임 준비
- CSS 애니메이션 또는 Canvas로 렌더링

### 2. 게임 월드 타일 시스템
- 타일 기반 맵 에디터
- 더 많은 타일 타입 추가
- 타일 배치 시스템

### 3. 게임 이벤트 트리거 개선
- 이벤트 발생 시 캐릭터가 실제로 이동하도록
- 이벤트 완료 시 보상 애니메이션
- 이벤트 실패 시 페널티 표시

## Unity 게임 추가 시

Unity 게임을 추가하면:
1. `public/game/unity-build/` 경로에 빌드 파일 배치
2. `CharacterGameLoader`가 자동으로 Unity 게임 로드
3. 2D 캔버스는 Unity 게임이 없을 때만 표시
