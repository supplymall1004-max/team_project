## 🎯 최적 배치 전략

### 1차 접근 경로 (가장 자주 사용)
```
/diet → "📸 식사 기록 & 분석" 탭
```
- **이유**: 식단 관리의 핵심 기능이므로 식단 페이지에 통합
- **사용자 여정**: 식단 확인 → 식사 사진 업로드 → 분석 확인

### 2차 접근 경로 (빠른 접근)
```
/diet → "오늘의 식단" 탭 → "📸 식사 기록" 버튼
```
- **이유**: 식단을 보다가 바로 식사 기록을 남기고 싶을 때
- **사용자 여정**: 오늘의 식단 확인 → 식사 기록 버튼 클릭

### 3차 접근 경로 (건강 관리 통합)
```
/health → "건강 기록" 탭 → "📸 식사 사진 분석" 카드
```
- **이유**: 건강 관리의 일환으로 식사 분석도 함께 관리
- **사용자 여정**: 건강 기록 확인 → 식사 사진 분석

## 📱 화면별 배치 상세

### `/diet` 페이지 - "📸 식사 기록 & 분석" 탭

#### 탭 구조
1. **사진 업로드** (기본 탭)
   - 날짜 선택
   - 아침/점심/저녁/간식별 사진 업로드
   - 사용 팁 제공

2. **식단 비교**
   - 날짜 선택
   - 추천 식단 vs 실제 식단 비교
   - 영양소별 차이 시각화

3. **주간 분석**
   - 주차 선택
   - 일주일간 영양소 분석 리포트
   - 부족/초과 영양소 알림

#### 사용 편의성 특징
- ✅ 한 화면에서 모든 기능 접근 가능
- ✅ 탭 전환으로 관련 기능 그룹화
- ✅ 날짜 선택이 각 탭에 있어 컨텍스트 유지

### `/health` 페이지 - "건강 기록" 탭

#### 카드 배치
- 왼쪽: 일반 건강 기록 (약물, 병원, 질병)
- 오른쪽: 식사 사진 분석 (강조된 카드)

#### 사용 편의성 특징
- ✅ 건강 관리의 일환으로 자연스럽게 통합
- ✅ 시각적으로 구분된 카드로 인지 용이
- ✅ 주요 기능 요약 제공

### "오늘의 식단" 섹션

#### 버튼 배치
- "건강 정보 관리" 버튼 옆에 "📸 식사 기록" 버튼 추가

#### 사용 편의성 특징
- ✅ 식단을 보다가 바로 식사 기록 가능
- ✅ 컨텍스트 전환 최소화
- ✅ 시각적으로 구분된 버튼 색상
