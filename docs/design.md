## 2. 주요 섹션 상세 설명

### 2.1. 헤더 (Header)
**위치**: 페이지 최상단, 모든 페이지에서 고정 표시

| 요소 | 설명 | 기능 |
| :--- | :--- | :--- |
| **로고** | "맛의 아카이브" 텍스트 또는 이미지 로고 | 클릭 시 홈으로 이동 |
| **메인 네비게이션** | 레거시 아카이브, 현대 레시피 북, AI 식단 큐레이션 | 각 섹션으로 이동 |
| **로그인/회원가입** | 사용자 인증 버튼 | 로그인 모달 또는 페이지 이동 |
| **프로필 아이콘** (로그인 시) | 사용자 프로필 메뉴 | 마이페이지, 설정, 로그아웃 |

**디자인 요구사항**:
- 모바일에서는 햄버거 메뉴(☰)로 축약
- 스크롤 시 헤더가 상단에 고정되도록 (Sticky Header)

---

### 2.2. 오늘의 추천 섹션 (Daily Picks)
**목적**: 방문 즉시 사용자에게 맞춤화된 콘텐츠를 제공하여 참여도 높이기

| 요소 | 설명 | 기능 |
| :--- | :--- | :--- |
| **슬로건** | "잊혀진 손맛을 연결하는 디지털 식탁" | 브랜드 아이덴티티 강조 |
| **메인 검색창** | 레시피 통합 검색 기능 | 키워드 입력 시 전체 레시피 검색 (A, B 섹션 통합) |
| **빠른 접근 버튼** | 3개의 주요 섹션으로 바로 이동 | 사용자 그룹별 맞춤 진입점 제공 |

**디자인 요구사항**:
- 각 카드에 호버 효과 적용 (살짝 확대 및 그림자)
- 별점과 인기도 표시로 신뢰성 강조
- 모바일에서는 스와이프 가능한 가로 카드 리스트

---

### 2.3. 빠른 검색 & 시작 섹션 (Quick Start)
**목적**: 서비스의 핵심 가치를 전달하고 즉시 사용을 유도

| 요소 | 설명 | 기능 ID (PRD 참조) |
| :--- | :--- | :--- |
| **섹션 제목** | "📜 레거시 아카이브" | - |
| **콘텐츠 카드 그리드** | 명인 인터뷰 영상 썸네일 4개 (가로 스크롤 또는 그리드) | A-1 (명인 인터뷰 뷰어) |
| **카드 정보** | 영상 제목, 명인 이름, 지역/시대 태그 | A-3 (아카이브 분류 검색) |
| **더보기 버튼** | 레거시 아카이브 전체 페이지로 이동 | A-1, A-2, A-3 전체 기능 |

**카드 구성 요소**:
- 썸네일 이미지 (16:9 비율)
- 영상 제목
- 명인 이름 및 지역 정보
- 재생 시간 표시
- 프리미엄 마크 (프리미엄 구독 전용 콘텐츠인 경우)

**디자인 요구사항**:
- 카드 호버 시 영상 미리보기 재생 (선택적)
- 모바일에서는 가로 스크롤 가능한 카드 리스트

---

### 2.4. 레거시 아카이브 & 현대 레시피 북 통합 섹션
**목적**: 일반 사용자에게 쉽고 빠른 레시피 검색 및 공유 기능 제공

| 요소 | 설명 | 기능 ID (PRD 참조) |
| :--- | :--- | :--- |
| **섹션 제목** | "🍴 현대 레시피 북" | - |
| **인기 레시피 카드** | 레시피 이미지, 별점, 제목, 조리시간, 난이도 | B-1 (단계별 레시피 카드), B-3 (평가 기능) |
| **레시피 업로드 버튼** | 사용자 레시피 등록 페이지로 이동 | B-2 (레시피 업로드 UGC) |
| **더보기 버튼** | 레시피 북 전체 페이지로 이동 | - |

**카드 구성 요소**:
- 레시피 대표 이미지 (1:1 또는 4:3 비율)
- 별점 (1~5점) 및 평가 개수
- 레시피 제목
- 조리 시간 및 난이도 표시
- 상업화 인증 마크 (인증된 레시피인 경우) - Post-MVP

**디자인 요구사항**:
- 카드 클릭 시 레시피 상세 페이지로 이동
- 별점은 시각적으로 명확하게 표시 (별 아이콘 사용)
- 모바일에서는 2열 그리드, 데스크톱에서는 4열 그리드

---

### 2.5. AI 맞춤 식단 큐레이션 & 건강 관리 섹션
**목적**: 건강 관리 식단 사용자에게 개인화된 식단 추천 및 식자재 구매 연동

| 요소 | 설명 | 기능 ID (PRD 참조) |
| :--- | :--- | :--- |
| **섹션 제목** | "🧠 AI 맞춤 식단 큐레이션" | - |
| **건강 정보 입력 버튼** | 사용자 건강 정보 입력 페이지로 이동 | C-1 (건강 정보 입력) |
| **오늘의 추천 식단** | 아침/점심/저녁 식단 카드 3개 | C-2 (AI 맞춤 식단 추천) |
| **식자재 구매 버튼** | 추천 식단의 재료를 한 번에 구매 | C-5 (식자재 원클릭 구매) |

**식단 카드 구성 요소**:
- 식사 시간 (아침/점심/저녁)
- 레시피 이미지
- 레시피 제목
- 칼로리 및 주요 영양소 정보 (칼로리, 탄수화물, 단백질, 지방)
- 레시피 바로가기 링크

**디자인 요구사항**:
- 로그인하지 않은 사용자에게는 "로그인 후 맞춤 식단을 확인하세요" 메시지 표시
- 건강 정보를 입력하지 않은 사용자에게는 "건강 정보 입력하기" 버튼 강조
- 프리미엄 구독자에게는 추가 혜택 배지 표시 (Post-MVP: C-4 영양 코칭 리포트)

---

### 2.6. 커뮤니티 & 소셜 섹션 (Community Hub)
**목적**: 사용자 간 소통과 콘텐츠 공유를 촉진하여 커뮤니티 형성

|| 요소 | 설명 | 기능 |
|| :--- | :--- | :--- |
|| **이주의 요리사** | 주간 인기 요리사 프로필 | 클릭 시 프로필 페이지로 이동 |
|| **요리 이야기** | 최근 커뮤니티 게시글 | 클릭 시 게시글 상세 페이지로 이동 |
|| **요리 사진 공유** | 인기 요리 사진 | 클릭 시 사진 상세 페이지로 이동 |

**디자인 요구사항**:
- 각 카드에 호버 효과 및 애니메이션 적용
- 사용자 참여도 표시 (좋아요, 댓글 수)
- 모바일 친화적인 터치 인터페이스

## 3. 모바일 반응형 디자인 가이드

### 3.1. 모바일 레이아웃 변경사항

| 요소 | 데스크톱 | 모바일 |
| :--- | :--- | :--- |
| **헤더** | 전체 메뉴 표시 | 햄버거 메뉴(☰)로 축약 |
| **히어로 섹션** | 가로 배치 버튼 3개 | 세로 배치 버튼 3개 (스택) |
| **레거시 아카이브** | 4열 그리드 | 가로 스크롤 카드 리스트 (2개씩 표시) |
| **레시피 북** | 4열 그리드 | 2열 그리드 |
| **AI 식단** | 가로 배치 3개 카드 | 세로 배치 3개 카드 (스택) |

### 3.2. 터치 친화적 요소
- 모든 버튼 최소 높이: 44px (iOS 가이드라인)
- 카드 간 간격: 16px 이상
- 텍스트 크기: 최소 14px (가독성 확보)

## 4. 색상 및 타이포그래피 가이드 (참고)

### 4.1. 주요 색상 (제안)
- **주 색상**: 따뜻한 주황/빨강 계열 (요리/음식 연상)
- **보조 색상**: 자연스러운 녹색 (건강/신선함 연상)
- **강조 색상**: 금색/노란색 (프리미엄 구독, 인증 마크)
- **텍스트**: 진한 회색 (#333333) 또는 검정 (#000000)
- **배경**: 밝은 베이지/크림색 (#FAFAFA)

### 4.2. 타이포그래피 (제안)
- **제목**: 굵은 서체 (Bold, 700)
- **본문**: 일반 서체 (Regular, 400)
- **강조 텍스트**: 중간 굵기 (Medium, 500)

## 5. 사용자 플로우 (User Flow)

### 5.1. 신규 사용자 (First-time Visitor)
```
홈페이지 진입
  ↓
오늘의 추천 섹션에서 관심 콘텐츠 발견
  ↓
빠른 검색 & 시작 섹션에서 서비스 이해
  ↓
[선택 1] 오늘의 레시피 둘러보기 → 레시피 상세 페이지
[선택 2] 명인 인터뷰 시청 → 레거시 아카이브 페이지
[선택 3] AI 식단 시작 → 로그인/회원가입 → 건강 정보 입력 → 맞춤 식단 확인
[선택 4] 통합 검색 사용 → 원하는 콘텐츠 검색 및 탐색
```

### 5.2. 기존 사용자 (Returning User)
```
홈페이지 진입
  ↓
자동 로그인 상태에서 개인화된 오늘의 추천 표시
  ↓
[선택 1] 오늘의 추천 콘텐츠 둘러보기
[선택 2] 통합 검색으로 원하는 콘텐츠 빠르게 찾기
[선택 3] AI 맞춤 식단 확인 및 조정
[선택 4] 커뮤니티에서 다른 사용자와 소통
  ↓
```

## 6. MVP 우선순위 기능 배치

## 7. 접근성 (Accessibility) 고려사항

- 모든 이미지에 대체 텍스트(Alt Text) 제공
