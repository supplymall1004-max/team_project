# 🏥 건강정보 공공 API 조사 결과

맛의 아카이브 프로젝트의 AI 맞춤 식단 큐레이션 기능(C-1, C-2)을 위해 개인 건강정보를 제공하는 공공 API를 조사한 결과입니다.

---

## 📋 조사 결과 요약

### ✅ 개인 건강정보를 제공하는 공공 API 존재

한국에는 개인의 건강정보를 조회할 수 있는 공공 API가 있습니다. 다만, **개인정보보호법**에 따라 사용자 본인의 동의와 인증이 필수입니다.

---

## 🔍 주요 공공 API 서비스

### 1. 건강정보 고속도로 (보건복지부)

**서비스 개요**:
- 2023년 9월부터 운영 시작
- 국가 중계 플랫폼으로 개인의 의료 데이터를 통합 관리
- 개인이 본인의 건강정보를 조회하고 활용할 수 있도록 지원

**제공 데이터**:
- 진료기록 (병원 방문 기록)
- 건강검진 결과 (혈압, 혈당, 콜레스테롤 등)
- 투약 정보 (처방전 정보)
- 예방접종 이력
- 건강 지표 (생활 습관, 식사 기록, 운동량 등)

**접근 방법**:
- "나의 건강기록" 앱을 통해 개인이 직접 조회
- API를 통한 연동 가능 (사용자 동의 필수)

**활용 가능성**: ⭐⭐⭐⭐
- 사용자가 직접 건강정보를 입력하는 대신, 공공 API를 통해 자동으로 가져올 수 있음
- 건강검진 결과를 기반으로 더 정확한 식단 추천 가능

---

### 2. 공공 마이데이터 서비스 (행정안전부)

**서비스 개요**:
- 개인이 자신의 공공기관 보유 정보를 직접 관리하고 활용
- 정보주체가 본인의 정보를 열람하고 제3자 제공 현황 확인 가능

**제공 데이터**:
- 건강보험 정보
- 연금 정보
- 납세 정보
- 보육 정보 등

**접근 방법**:
- 공공 마이데이터 유통 시스템을 통한 API 연동
- 사용자 동의 및 인증 필수

**활용 가능성**: ⭐⭐⭐
- 건강보험 정보를 통해 간접적으로 건강 상태 파악 가능
- 직접적인 건강검진 결과는 제공하지 않을 수 있음

---

### 3. 국민건강보험공단 API

**서비스 개요**:
- 건강보험 가입자 정보 조회
- 건강검진 결과 조회 (제한적)

**제공 데이터**:
- 건강보험 가입 정보
- 건강검진 결과 (일부)
- 진료비 정보

**접근 방법**:
- 공공데이터포털(data.go.kr)을 통한 API 신청
- 사업자 등록 및 승인 필요

**활용 가능성**: ⭐⭐⭐
- 건강검진 결과를 직접 가져올 수 있지만, 승인 절차가 복잡할 수 있음

---

## ⚠️ 법적 제약사항 및 주의사항

### 1. 개인정보보호법 준수

**필수 사항**:
- 사용자의 **명시적 동의** 필수
- 건강정보는 **민감정보**로 분류되어 더 엄격한 보호 필요
- 정보 수집 목적 명시 및 동의 범위 내에서만 사용

**PRD 요구사항과의 일치**:
- PRD 4장 비기능 요구사항에서 "개인 건강 정보는 최고 수준으로 암호화" 명시
- PRD 6.6 면책 조항: "건강 관리 보조 수단"임을 명시

### 2. API 연동 시 고려사항

**기술적 요구사항**:
- OAuth 2.0 또는 공인인증서 기반 인증
- 사용자 본인 인증 필수 (휴대폰 인증, 공동인증서 등)
- API 호출 시 사용자 동의 토큰 필요

**보안 요구사항**:
- 건강정보 암호화 저장 (PRD C-1 요구사항)
- 전송 중 데이터 암호화 (HTTPS 필수)
- 접근 로그 기록 및 감사

## 💡 맛의 아카이브 프로젝트 적용 방안

### 방안 1: 하이브리드 접근 (권장) ⭐⭐⭐⭐⭐

**구조**:
```
사용자 선택
  ├─ [옵션 1] 공공 API 연동 (자동 입력)
  │   └─ 건강정보 고속도로 API 연동
  │   └─ 사용자 인증 후 최근 건강검진 결과 자동 가져오기
  │
  └─ [옵션 2] 수동 입력 (기본)
      └─ 사용자가 직접 알레르기, 질병 정보 입력
```

**장점**:
- 사용자 편의성 향상 (자동 입력)
- 데이터 정확도 향상 (건강검진 결과 기반)
- 사용자가 원하지 않으면 수동 입력 가능

**구현 단계**:
1. **MVP 단계**: 수동 입력만 구현 (C-1 기본 기능)
2. **Post-MVP**: 공공 API 연동 추가 (고도화)

### 방안 2: 공공 API 우선 접근

**구조**:
- 공공 API를 기본으로 하고, API에서 제공하지 않는 정보만 수동 입력

**단점**:
- API 승인 절차가 오래 걸릴 수 있음
- 모든 사용자가 공공 API 사용 가능한 것은 아님 (미가입자 등)
- MVP 개발 지연 가능

## 🎯 권장 구현 전략

### Phase 1: MVP (현재 계획 유지)

**C-1 기능**: 사용자 수동 입력
- 알레르기 정보
- 질병 정보 (당뇨, 고혈압 등)
- 선호/비선호 식재료

**이유**:
- 개발 속도 빠름
- 법적 리스크 최소화
- 모든 사용자에게 동일한 기능 제공

### Phase 2: Post-MVP (고도화)

**공공 API 연동 추가**:
- 건강정보 고속도로 API 연동
- 사용자 선택: "건강검진 결과 자동 가져오기" 옵션
- 최근 건강검진 결과를 기반으로 자동으로 알레르기/질병 정보 추출

**추가 기능**:
- 건강검진 결과 기반 영양소 목표 자동 설정
- 검진 결과 변화 추이 모니터링 (선택적)

## 📝 API 연동 시 필요한 작업

### 1. 사업자 등록 및 승인

- 공공데이터포털(data.go.kr) 사업자 등록
- API 사용 신청 및 승인 대기
- 예상 소요 시간: 1-2주

### 2. 기술적 구현

**필수 구현 사항**:
- OAuth 2.0 인증 플로우
- 사용자 본인 인증 (휴대폰 인증 등)
- API 클라이언트 구현
- 에러 처리 및 재시도 로직
- 데이터 암호화 저장

**예상 개발 시간**: 1-2주

### 3. 법적 문서 준비

- 개인정보 처리방침 업데이트
- 이용약관 업데이트 (건강정보 수집 동의 조항)
- API 제공사와의 계약 체결
