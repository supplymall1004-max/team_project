# 콘솔 에러 확인 가이드

## 브라우저 개발자 도구로 콘솔 에러 확인하기

### 1. 개발 서버 실행 확인
```bash
# 터미널에서 확인
pnpm dev
```

### 2. 브라우저에서 개발자 도구 열기
- **Windows/Linux**: `F12` 또는 `Ctrl + Shift + I`
- **Mac**: `Cmd + Option + I`

### 3. Console 탭 확인
1. 개발자 도구가 열리면 **Console** 탭 클릭
2. 빨간색 에러 메시지 확인
3. 에러 메시지의 전체 내용 복사

### 4. Network 탭 확인 (필요시)
1. **Network** 탭 클릭
2. 페이지 새로고침 (`F5`)
3. 실패한 요청(빨간색) 확인
4. 실패한 요청 클릭하여 상세 정보 확인

## 예상되는 에러 유형

### 1. 정적 파일 로더 관련 에러
**에러 메시지 예시:**
```
[StaticRecipeLoader] 레시피 디렉토리가 없습니다: ...
```

**원인**: `docs/mfds-recipes/recipes/` 디렉토리가 없음

**해결 방법**: 
- 정상적인 상황입니다 (아직 데이터 수집이 안 됨)
- API 폴백이 작동하므로 문제 없음

### 2. fs 모듈 관련 에러
**에러 메시지 예시:**
```
Module not found: Can't resolve 'fs'
```

**원인**: 클라이언트 컴포넌트에서 서버 전용 모듈 사용

**해결 방법**: 
- `loadAllStaticRecipes`는 Server Component에서만 사용되어야 함
- 이미 Server Component에서만 사용 중이므로 문제 없음

### 3. API 호출 타임아웃 에러
**에러 메시지 예시:**
```
TimeoutError: The operation was aborted due to timeout
```

**원인**: 식약처 API 응답 지연

**해결 방법**: 
- 정적 파일 시스템이 활성화되면 해결됨
- 현재는 API 폴백이 작동하므로 문제 없음

## 현재 구현 상태

### ✅ 정상 작동하는 부분
1. **정적 파일 로더**: 서버 사이드에서만 실행됨
2. **에러 처리**: try-catch로 모든 에러 처리됨
3. **폴백 로직**: 정적 파일이 없으면 API로 폴백

### ⚠️ 확인이 필요한 부분
1. **실제 브라우저 콘솔 에러**: 직접 확인 필요
2. **서버 사이드 에러**: 터미널 로그 확인 필요

## 에러 보고 방법

콘솔에서 에러를 발견하면 다음 정보를 포함하여 보고해주세요:

1. **에러 메시지 전체 내용**
2. **에러가 발생한 파일명과 줄 번호**
3. **에러가 발생한 페이지 URL**
4. **브라우저 종류 및 버전**
5. **스크린샷 (가능한 경우)**
