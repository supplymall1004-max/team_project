# 반응형 디자인 테스트 가이드

맛의 아카이브 프로젝트의 반응형 디자인 테스트 가이드입니다.

## 1. 테스트 환경

### 브라우저 개발자 도구

#### Chrome DevTools
1. F12 또는 우클릭 → 검사
2. 디바이스 툴바 토글 (Ctrl+Shift+M)
3. 반응형 모드 선택

#### Firefox DevTools
1. F12 또는 우클릭 → 요소 검사
2. 반응형 디자인 모드 (Ctrl+Shift+M)

### 실제 디바이스 테스트

#### 모바일 디바이스
- **iOS**: iPhone SE (375px), iPhone 12/13 (390px), iPhone 14 Pro Max (430px)
- **Android**: Galaxy S21 (360px), Pixel 5 (393px)

#### 태블릿 디바이스
- **iPad**: 768px × 1024px
- **iPad Pro**: 1024px × 1366px

#### 데스크톱
- **작은 화면**: 1280px × 720px
- **표준 화면**: 1920px × 1080px
- **큰 화면**: 2560px × 1440px

## 7. 실제 디바이스 테스트 방법

### 로컬 네트워크 테스트

1. **개발 서버 실행**
   ```bash
   pnpm dev
   ```

2. **로컬 IP 확인**
   ```bash
   # Windows
   ipconfig
   
   # Mac/Linux
   ifconfig
   ```

3. **모바일 디바이스에서 접속**
   - 같은 Wi-Fi에 연결
   - 브라우저에서 `http://[로컬IP]:3000` 접속

### Chrome Remote Debugging

1. **모바일에서 Chrome 원격 디버깅 활성화**
   - Chrome → 설정 → 개발자 도구 → 원격 디버깅

2. **데스크톱 Chrome에서 연결**
   - `chrome://inspect` 접속
   - 연결된 디바이스 선택

## 8. 자동화 테스트

### Playwright 테스트 예시

```typescript
import { test, expect } from '@playwright/test';

test('모바일 레이아웃 테스트', async ({ page }) => {
  await page.setViewportSize({ width: 375, height: 667 });
  await page.goto('/');
  
  // 헤더가 햄버거 메뉴로 변경되는지 확인
  const hamburger = page.locator('[aria-label="메뉴"]');
  await expect(hamburger).toBeVisible();
  
  // 카드가 1열로 표시되는지 확인
  const cards = page.locator('[data-testid="recipe-card"]');
  const firstCard = cards.first();
  const secondCard = cards.nth(1);
  
  const firstRect = await firstCard.boundingBox();
  const secondRect = await secondCard.boundingBox();
  
  // 두 카드가 세로로 배치되어 있는지 확인
  expect(secondRect!.y).toBeGreaterThan(firstRect!.y + firstRect!.height);
});
```

## 9. 문제 해결

### 일반적인 문제

#### 레이아웃 깨짐
- **원인**: 고정 너비 사용
- **해결**: `max-w-*` 클래스 사용, `w-full` 대신 `flex-1` 사용

#### 텍스트 오버플로우
- **원인**: 긴 텍스트 처리 없음
- **해결**: `truncate`, `line-clamp-*` 클래스 사용

#### 이미지 비율 깨짐
- **원인**: aspect-ratio 미지정
- **해결**: `aspect-*` 클래스 사용
