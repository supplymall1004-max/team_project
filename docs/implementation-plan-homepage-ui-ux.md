## 1. 개요

### 1.1 목표

배달의민족 앱의 사용자 경험을 참고하여 Flavor Archive 홈페이지를 다음과 같이 개선:

- **상단 고정 영역**: 검색바와 프리미엄 배너를 항상 접근 가능하게
- **바로가기 메뉴**: 주요 기능을 아이콘 그리드로 빠른 접근
- **하단 네비게이션**: 주요 페이지로의 쉬운 이동
- **개인화 콘텐츠**: 주간 식단 요약, 자주 구매하는 식자재 섹션

### 1.2 핵심 원칙

1. **고정/스크롤 영역 분리**: 중요한 CTA는 항상 보임
2. **빠른 접근성**: 자주 사용하는 기능은 1-2클릭으로 접근
3. **개인화**: 사용자 데이터 기반 맞춤 콘텐츠 제공
4. **모바일 우선**: 모바일 환경에서 최적화된 UX

**목표**: 고정 헤더와 하단 네비게이션 구현

4. 하단 네비게이션 컴포넌트 생성
5. 홈페이지 레이아웃 수정 (고정/스크롤 분리)

**예상 소요 시간**: 5-7일

### Phase 2: 바로가기 메뉴 (3일)

**목표**: 아이콘 그리드 형태의 바로가기 메뉴 구현

**작업 내용**:
1. 바로가기 메뉴 컴포넌트 생성
2. 아이콘 선택 및 스타일링
3. 반응형 그리드 레이아웃
4. 호버/터치 효과

**예상 소요 시간**: 2-3일

### Phase 3: 콘텐츠 섹션 (1주)

**목표**: 주간 식단 요약 및 자주 구매하는 식자재 섹션 구현

**작업 내용**:

2. 자주 구매하는 식자재 섹션 컴포넌트
3. API 엔드포인트 구현 (`/api/shopping/frequent-items`)
4. 추천 레시피 섹션 개선

**예상 소요 시간**: 5-7일

### Phase 4: 세부 개선 (3일)

**목표**: 애니메이션, 접근성, 성능 최적화

**작업 내용**:
1. 애니메이션 및 호버 효과
2. 모바일 최적화 테스트
3. 접근성 검증 (키보드 네비게이션, 스크린리더)
4. 성능 최적화 (이미지 lazy loading, 코드 스플리팅)

// FixedHeader
interface FixedHeaderProps {

### 5.2 API 엔드포인트

#### GET /api/shopping/frequent-items

**요청**: 인증 필요 (Clerk 세션)

**응답**:
```json
{
  "items": [
    {
      "id": "item-1",
      "name": "양파",
      "imageUrl": "https://...",
      "price": 3000,
      "category": "채소"
    },
    ...
  ]
}
```

**로직**:
1. 사용자 ID 조회 (Clerk)
2. 최근 4주간의 주간 식단 조회
